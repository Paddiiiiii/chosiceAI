基于《作战指挥手册》内容，以**（二）组织侦察情报**为贯穿示例，提供各场景的接口调用流程、测试用例及期望结果。

> 各接口的**功能**、**使用场景**、**概述**详见 [API接口文档.md](./API接口文档.md)。

## 一、贯穿示例：组织侦察情报

与 `技术方案_智能体知识服务.md` 中图谱示例一致，以「组织侦察情报」为锚点串联各接口。

### 图谱结构（参考）

```
Phase: 战斗准备
    │ BELONGS_TO
    ▼
TaskGroup: （二）组织侦察情报  chunk_id: ch1_s01_p01_t02
    │ HAS_SUBTASK
    ├── Task: 1.确定情报需求   (ch1_s01_p01_t02_01)
    ├── Task: 2.概算战术目标   (ch1_s01_p01_t02_02)
    ├── Task: 3.制定侦察计划   (ch1_s01_p01_t02_03)  ← LED_BY:侦察情报要素, APPROVED_BY:参谋长
    └── Task: 4.组织侦察协同   (ch1_s01_p01_t02_04)
         │ NEXT_STEP (顺序)
         │ PRODUCES → Product: 侦察计划
```

### chunk_id 如何获得

| 方式 | 说明 |
|------|------|
| **Search 前 5 条** | `POST /search/comparison` 的 `rrf_results` 中，取前 5 条的 `chunk_id`，分别调用后续接口并融合结果 |
| **Resolve 返回** | `POST /resolve` 的 `matches` 中，每条含 `chunk_id` |
| **查表** | 见文末「Chunk ID 参考」表，已知任务时可直接使用 |

## 三、场景 1：任务拆解

**目标**：将大任务（自然语言）拆成可执行的小任务/子步骤。

### 调用流程

```
1. POST /search/comparison  query="组织侦察情报的流程"
   → 取 rrf_results 前 5 条的 chunk_id 列表

2. POST /resolve  query="组织侦察情报的流程"  （可选，精确定位）

3. 对每个 chunk_id 调用 GET /graph/task_decompose
   → 融合、去重后得到完整子步骤列表
```

### 组织侦察情报用例

| 步骤 | 接口 | 入参 | 期望结果 |
|------|------|------|----------|
| 1 | POST /search/comparison | `{"query":"组织侦察情报的流程", "retrieval":{...}}` | rrf_results 首条 chunk_id 为 ch1_s01_p01_t02 或相关 |
| 2 | GET /graph/task_decompose | `chunk_id=ch1_s01_p01_t02` | 4 个子步骤：1.确定情报需求、2.概算战术目标、3.制定侦察计划、4.组织侦察协同 |

### 入参示例

**Step 1 - Search**
```json
POST /api/v1/search/comparison
{
  "query": "组织侦察情报的流程",
  "filters": null,
  "retrieval": { "use_vector": true, "use_bm25": true, "use_graph": true }
}
```

**Step 3 - Task Decompose**（chunk_id 来自 Step 1 的 rrf_results[0].chunk_id）
```
GET /api/v1/graph/task_decompose?chunk_id=ch1_s01_p01_t02
```

## 四、场景 2：找对应人员

**目标**：查某任务由谁负责、涉及哪些角色。

### 调用流程

```
方式 A：按任务查角色
  1. POST /search/comparison  query="制定侦察计划"
  2. GET  /graph/task_roles?chunk_id={chunk_id}

方式 B：按角色查任务
  GET  /graph/role_tasks?role=侦察情报要素&phase=战斗准备
```

### 组织侦察情报用例

| 接口 | 入参 | 期望结果 |
|------|------|----------|
| task_roles | chunk_id=ch1_s01_p01_t02_03（3.制定侦察计划） | led_by: 侦察情报要素；approved_by: 参谋长（若 LLM 抽取） |
| role_tasks | role=参谋长, phase=战斗准备 | 含「制定侦察计划」审批等任务 |
| role_tasks | role=侦察情报要素 | 含「制定侦察计划」拟制等任务 |

### 入参示例

```
GET /api/v1/graph/task_roles?chunk_id=ch1_s01_p01_t02_03
GET /api/v1/graph/role_tasks?role=参谋长&phase=战斗准备
```

## 七、场景 5：路由判定（一站式）

**目标**：旅长说自然语言，系统直接返回牵头负责角色及依据。

### 组织侦察情报相关用例

| 输入（input） | 期望结果 |
|---------------|----------|
| 侦察计划该谁去做？ | lead: 侦察情报要素 或 参谋长；reasoning 引用「通常由侦察情报要素负责拟制侦察计划，报参谋长审批」 |

### 入参示例

```json
POST /api/v1/chat
{
  "input": "侦察计划该谁去做？",
  "context": { "phase": "战斗准备" },
  "retrieval": { "use_vector": true, "use_bm25": true, "use_graph": true }
}
```

## 八、可执行脚本

项目提供 `scripts/test_api_flows.py`，支持两种模式：

```bash
# 贯穿示例：以「组织侦察情报」串联 5 个场景
python scripts/test_api_flows.py --demo

# 按场景单独运行
python scripts/test_api_flows.py --scene 1
python scripts/test_api_flows.py --scene 2 --role 参谋长 --phase 战斗准备
python scripts/test_api_flows.py --scene 3 --chunk-id ch1_s01_p01_t02_03
python scripts/test_api_flows.py --scene 4 --chunk-id ch1_s01_p01_t02_03
python scripts/test_api_flows.py --scene 5

# 运行全部场景（通用用例）
python scripts/test_api_flows.py
```
